# 验收与证据规则

## 1. 验收项必须可判定

软验收：

```text
- 功能基本可用
- 页面正常展示
- 逻辑完善
- 测试通过
```

硬验收：

```text
- 当输入缺少 required 字段时，CLI 返回非 0 状态码，并输出包含字段名的错误信息。
- 新增 tests/test_contract_builder.py，覆盖正常输入、空输入、重复 memory id、非法 schema 四种场景。
- 运行 `pytest tests/test_contract_builder.py` 通过。
- CLI 输出的 contract 文件可被 verifier 读取，字段包含 obligations、source_memories、scope、created_at。
```

## 2. 每个验收项都要绑定证据

证据类型包括：

- 测试文件路径
- 运行命令
- 命令输出摘要
- 修改文件路径
- 日志片段
- 截图
- 生成产物路径
- 手动验证步骤

## 3. 反偷懒验收必须固定存在

每个任务都要有：

```markdown
## 反偷懒验收

- [ ] 没有未经授权降级成 MVP。
- [ ] 没有 TODO / placeholder / mock 冒充完成。
- [ ] 没有只做 happy path。
- [ ] 没有只写接口不接入真实调用链。
- [ ] 没有只写文档不实现。
- [ ] 每个验收项都有证据。
```

## 4. 长任务证据回填

最终完成报告必须包含：

```markdown
## Evidence Report

### 修改文件

### 新增/更新测试

### 运行命令

### 命令结果

### 验收项到证据映射

| 验收项 | 证据 | 是否通过 |
|---|---|---|

### 未覆盖风险

### TODO / placeholder / mock 检查
```

## 5. 不允许的完成声明

以下表达不能作为完成证据：

- “应该可以”
- “基本完成”
- “逻辑已完善”
- “后续可以补充”
- “已预留接口”
- “理论上支持”
- “未实际运行，但代码看起来没问题”
