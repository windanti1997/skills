## 11. 反偷懒验收

- [ ] 没有把完整目标降级成未经授权的 MVP。
- [ ] 没有用 TODO / placeholder / mock 冒充完成。
- [ ] 没有只实现 happy path。
- [ ] 没有只写接口而不接入真实调用链。
- [ ] 没有只更新文档而不完成代码，除非本任务明确是文档任务。
- [ ] 每个验收项都有证据来源。
- [ ] 允许后置项不会破坏本次闭环。
- [ ] 禁止降级项全部满足。
- [ ] 测试覆盖范围明确，不是只写"有测试"。
- [ ] 单测覆盖了核心逻辑和边界，不是只测表面。
- [ ] 集成测试覆盖了模块交互，不是只有单元测试孤岛。

## 14. Review 打回条件

出现以下任一情况必须打回：

- 任务被降级成未授权 MVP。
- 有 TODO / placeholder / mock 冒充完成。
- 没有测试或证据。
- 只覆盖 happy path。
- 没有接入真实调用链。
- 验收标准无法独立判断。
- 与用户真实目标不一致。
- 缺少边界场景。
- 允许后置项实际破坏本次闭环。
- 测试覆盖范围不明确，只写了"有测试"三个字。
- 单测只测表面现象，不测核心逻辑。
- 多模块任务没有集成测试方案，只有单元测试孤岛。
- 多模块任务没有联调集成测试任务。
