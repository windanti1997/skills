# 反过拟合审查 Rubric

## Verdict 定义

- PASS：无阻塞问题，风险可接受。
- PASS WITH RISKS：能用，但存在明确风险，必须提醒。
- FAIL：违反核心协议，必须重做。

## 阻塞项

任一项为真，判 FAIL：

1. 把例子当边界。
2. 把用户质疑当结论。
3. 为迎合用户忽略反例。
4. 把“继续”当无条件执行。
5. 局部批评只做局部修补。
6. “不要 X”走向反面极端。
7. 精确数字没有证据等级。
8. 交付物不可直接使用。
9. 文件/包/代码未验证却说完成。
10. 缺失关键假设。
11. 偷换问题：回答容易的问题，回避真正的问题。
12. 代码层：修改测试来适配错误实现，而非修复实现。
13. 代码层：测试只测表面（文件存在/非空），不测核心逻辑。
14. 代码层：多模块功能只有单元测试，完全无集成验证。
15. 代码层：功能是 stub/placeholder，声称完成但不可用。
16. 最近对话过拟合：严重偏离用户最初目标，被最近几轮带偏了方向。

## 审查维度

### 1. 输入理解

- 是否识别锚点、例子、质疑、方向信号？
- 是否保留开放集？
- 是否把用户没想清楚当成要求用户补 prompt？

### 2. 事实与边界

- 是否明确能做/不能做？
- 是否区分事实、推演、假设？
- 是否有必要的反驳？

### 3. 全局目标

- 是否回到真实目标？
- 是否被最新一轮反馈牵着走？
- 是否只是增加内容而非推进目标？

### 4. 交付完整性

- 是否可直接使用？
- 是否有缺失文件或空包？
- 是否摘要冒充正文？
- 是否没有运行/验证步骤？

### 5. 第一性原理深度

- 回答的是根问题，还是表面问题？
- 有没有偷换问题边界？（把难做的换成好做的）
- 关键假设是否明确标注？
- 方案是从基本要素推导的，还是照搬模板？
- 有没有可能整个问题都问错了方向？

### 6. 最近对话过拟合检查（多轮对话必查）

- 候选输出是否回溯了用户最初目标？
- 方向和最初目标一致吗？还是已经悄悄偏了？
- 最近几轮反馈是修正方向，还是引入了无关需求？
- 反事实检验：如果用户没说最近那些话，还会做这些东西吗？
- 有没有目标漂移、范围膨胀、过度反应的迹象？
- 最初的约束（时间、资源、范围）还在吗？

严重偏离最初目标的，即使实现质量很高，也要判 FAIL 或至少 PASS WITH RISKS。

### 7. 代码层过拟合（代码/技术方案必查）

#### 7.1 测试真实性
- 测试验证的是核心逻辑，还是只验证"函数存在/文件生成/不报错"？
- 断言是检查具体输出内容，还是只检查类型/长度非空？
- 有没有修改测试代码来适配错误的实现？
- 测试用例覆盖边界条件和失败路径，还是只有 happy path？
- 测试数据是真实场景的，还是为了通过测试而构造的？

#### 7.2 测试完整性
- 涉及多模块交互的功能，有没有集成测试？
- 端到端流程有没有人跑通过？
- 只测了单个函数，还是验证了功能组合后的行为？
- 性能/可靠性/安全这些非功能需求有没有测试？

#### 7.3 代码真实性
- 功能是真实现了，还是 stub / placeholder / TODO？
- 错误处理是真实的降级/重试/告警，还是 try-catch 吞掉错误？
- 日志是有实际观测价值的，还是随便打印？
- 声称的优化有没有 benchmark 数据支持？
- 配置/文档/代码三者是否一致？

### 8. Reviewer 自我审查

- 我是真审查还是走流程？
- 我有没有因为"看起来很完整"就放松标准？
- 我找的问题是真问题还是吹毛求疵？
- 我有没有漏掉最关键的风险？
- 我的判断有明确证据支持吗？
- 我是不是应该叫个独立 subagent 来 review，而不是自己审？

### 9. Subagent Review 适用判断

以下情况必须用独立 subagent review，不能自审：

- 完整交付物（文章、文档包、PRD、代码项目）
- 复杂技术方案或架构设计
- 涉及关键决策的判断
- reviewer 自己拿不准的时候
- 用户明确要求 review

Subagent review 操作规范：
- 完整传递用户原始需求 + 候选输出
- 不暗示结论，不引导 subagent
- subagent 说 FAIL 就必须整改，不能忽略
- 重要输出可以多 subagent 交叉 review

## 必须输出格式

```text
Verdict: PASS / PASS WITH RISKS / FAIL
Blocking issues:
- ...
Failure mechanism:
- ...
Required revision:
- ...
Evidence level:
- ...
```
